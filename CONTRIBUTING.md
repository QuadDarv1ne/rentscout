# 📝 CONTRIBUTING.md

## Как внести вклад в RentScout

Мы рады любому вкладу в развитие проекта! Вот руководство, которое поможет вам начать.

## 🚀 Быстрый старт

### 1. Форк и клонирование

```bash
git clone https://github.com/ваш-username/rentscout.git
cd rentscout
```

### 2. Настройка окружения

```bash
# Создание виртуального окружения
python -m venv venv
source venv/bin/activate  # На Windows: venv\Scripts\activate

# Установка зависимостей
make install
make install-dev
```

### 3. Установка pre-commit hooks

```bash
make pre-commit
```

## 🔧 Процесс разработки

### Создание новой ветки

```bash
git checkout -b feature/your-feature-name
# или
git checkout -b fix/bug-description
```

### Именование веток

- `feature/` - новая функциональность
- `fix/` - исправление багов
- `docs/` - изменения в документации
- `refactor/` - рефакторинг кода
- `test/` - добавление тестов

### Написание кода

1. **Следуйте стилю кода:**
   ```bash
   make format  # Автоформатирование
   make lint    # Проверка линтерами
   ```

2. **Пишите тесты:**
   ```bash
   make test       # Запуск тестов
   make test-cov   # С покрытием
   ```

3. **Документируйте код:**
   - Docstrings для функций и классов
   - Комментарии для сложной логики
   - Обновление README.md при необходимости

## ✅ Checklist перед коммитом

- [ ] Код отформатирован (`make format`)
- [ ] Линтеры проходят (`make lint`)
- [ ] Тесты проходят (`make test`)
- [ ] Добавлены новые тесты для новой функциональности
- [ ] Документация обновлена
- [ ] Коммит-сообщения понятны и информативны

## 📝 Коммит-сообщения

Используйте Conventional Commits:

```
feat: добавить поддержку парсера Sutochno
fix: исправить утечку памяти в кеше Redis
docs: обновить API документацию
test: добавить тесты для AvitoParser
refactor: оптимизировать SearchService
```

## 🔍 Pull Request

### Создание PR

1. Убедитесь, что ваша ветка обновлена:
   ```bash
   git fetch origin
   git rebase origin/master
   ```

2. Создайте Pull Request в GitHub

3. Заполните шаблон PR:
   - Описание изменений
   - Связанные Issues
   - Скриншоты (если применимо)
   - Чек-лист

### Критерии принятия PR

- ✅ Все тесты проходят
- ✅ Покрытие кода не уменьшилось
- ✅ Линтеры проходят без ошибок
- ✅ Код ревью одобрен
- ✅ Документация обновлена

## 🧪 Тестирование

### Запуск тестов локально

```bash
# Все тесты
make test

# С покрытием
make test-cov

# Конкретный файл
pytest app/tests/test_avito_parser.py -v

# Конкретный тест
pytest app/tests/test_avito_parser.py::test_parse_success -v
```

### Добавление тестов

- Используйте `pytest` и `pytest-asyncio`
- Мокируйте внешние API
- Тестируйте edge cases
- Стремитесь к 80%+ покрытию

## 🔒 Безопасность

### GitHub Secrets для CI/CD

Для работы CI/CD pipeline необходимо настроить следующие секреты в настройках репозитория:

| Secret | Описание | Пример |
|--------|----------|--------|
| `SECRET_KEY` | Секретный ключ для подписи токенов | `your-secret-key-min-32-chars` |
| `JWT_SECRET` | Секретный ключ для JWT | `your-jwt-secret-min-32-chars` |
| `TEST_DB_PASSWORD` | Пароль для тестовой БД | `secure_test_password` |

**Важно:** В CI/CD используются fallback значения по умолчанию для тестирования, но для production рекомендуется установить собственные секреты.

### Сообщение об уязвимостях

Если нашли уязвимость безопасности:

1. **НЕ** создавайте публичный Issue
2. Напишите на security@rentscout.dev
3. Опишите уязвимость и шаги воспроизведения

## 🔴 Критические (High Priority)

### 1. Отсутствуют миграции БД в CI
- **Проблема:** В `.github/workflows/ci.yml` нет шага `alembic upgrade head`
- **Решение:** Добавить шаг миграций перед тестами
- **Время:** 1-2 часа
- **Сложность:** Низкая
- **Влияние:** Высокое
- **Чеклист:**
  - [ ] Добавить шаг миграций в ci.yml
  - [ ] Протестировать локально
  - [ ] Закоммитить

### 2. Нет секретов для production
- **Проблема:** В `.env.example` слабые дефолтные значения
- **Решение:** Добавить `scripts/generate_secrets.py`
- **Время:** 30 минут
- **Сложность:** Низкая
- **Влияние:** Высокое
- **Чеклист:**
  - [ ] Создать scripts/generate_secrets.py
  - [ ] Добавить генерацию SECRET_KEY, JWT_SECRET, POSTGRES_PASSWORD, REDIS_PASSWORD
  - [ ] Обновить README

### 3. Слабая обработка ошибок парсеров
- **Проблема:** Парсеры могут падать без recovery
- **Решение:** Добавить circuit breaker в каждый парсер
- **Время:** 2-3 часа
- **Сложность:** Средняя
- **Влияние:** Высокое
- **Чеклист:**
  - [ ] Применить CircuitBreaker к Avito парсеру
  - [ ] Применить к Cian парсеру
  - [ ] Применить к остальным парсерам
  - [ ] Добавить тесты

### 4. Нет rate limiting для внешних API
- **Проблема:** Риск блокировки со стороны Avito/Cian
- **Решение:** Добавить адаптивный rate limiting
- **Время:** 3-4 часа
- **Сложность:** Средняя
- **Влияние:** Среднее
- **Чеклист:**
  - [ ] Создать app/utils/adaptive_ratelimit.py
  - [ ] Интегрировать в базовый парсер
  - [ ] Добавить логику увеличения задержки при 429/403
  - [ ] Тесты

### 5. Тесты используют SQLite
- **Проблема:** В production PostgreSQL, возможны различия в поведении
- **Решение:** Запускать часть тестов на реальном PostgreSQL
- **Время:** 2-3 часа
- **Сложность:** Средняя
- **Влияние:** Высокое
- **Чеклист:**
  - [ ] Добавить fixture с PostgreSQL в docker-compose.ci.yml
  - [ ] Обновить conftest.py
  - [ ] Разделить тесты на SQLite/PostgreSQL
